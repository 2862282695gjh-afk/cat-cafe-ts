# 短期记忆（CLI Session）

## 概念

每只猫咪通过 Claude CLI 的 `--resume {sessionId}` 机制维持对话上下文。CLI 进程内部保存了完整的对话历史，猫咪"记得"之前几轮说了什么。

## 核心文件

| 文件 | 职责 |
|------|------|
| `packages/provider-claude/src/claude-process.ts` | CLI 进程管理：启动、发送、恢复、切换 |
| `packages/server/src/store/session-store.ts` | session ID 持久化到 `data/sessions.json` |
| `packages/server/src/ws/handler.ts` | 调用时读取 session、完成后保存 session |

## sessions.json 结构

```json
{
  "agentId:threadId": "cliSessionId"
}
```

示例：

```json
{
  "tamako:bed0024c-bd3e-...": "ba511c84-d82e-...",
  "sasaki:bed0024c-bd3e-...": "4018d337-5f3e-...",
  "bunzo:bed0024c-bd3e-...": "7fd41b59-38de-...",
  "kohana:bed0024c-bd3e-...": "23fc715f-0917-..."
}
```

- **key**：`猫咪ID : 对话线程ID`，O(1) 查询
- **value**：Claude CLI 的 session UUID，用于 `--resume` 恢复上下文

## 完整流程

### 1. 用户发送消息

```
用户在对话 bed0024c 发消息 → 前端带上 threadId → 后端路由给社珠子
```

### 2. 查找 session

```typescript
// handler.ts
const threadSession = getSessionId(agentId, threadId);  // 查 sessions.json
claudeAgent.switchSession(threadSession);                // 切到对应 session
```

### 3. switchSession 逻辑

```typescript
switchSession(newSessionId: string | null): void {
  if (this._sessionId === newSessionId && this._alive) return;  // 同一个，不用切
  this.killProcess();           // 杀掉当前 CLI 进程
  this._sessionId = newSessionId;  // 设为新 session ID
}
```

### 4. ensure() 启动进程

```typescript
ensure(): Promise<void> {
  // 如果进程还活着，直接复用
  if (this.proc && this._alive) return;

  // 否则启动新进程
  const args = ["--output-format", "stream-json", "--input-format", "stream-json"];

  if (this._sessionId) {
    args.push("--resume", this._sessionId);  // 恢复旧上下文
  }
  // 启动 claude 子进程
}
```

### 5. 完成后保存

```typescript
// result 事件中
agent.sessionId = resultEvent.session_id;
saveSessionId(agentId, threadId, resultEvent.session_id);  // 写入 sessions.json
```

## 进程生命周期

| 场景 | 进程状态 | session ID |
|------|---------|------------|
| 同一线程连续对话 | 存活，直接复用 | 不变 |
| 切换到另一个对话 | 被杀掉 | 切换到目标线程的 session ID |
| 程序重启 | 进程不存在 | 从 sessions.json 恢复，首次调用时启新进程 + --resume |
| 上下文压缩后 | 被杀掉 | 清为 null，下次生成新 UUID |

## 关键设计点

- **线程隔离**：每只猫在每个对话里都有独立的 CLI session，互不干扰
- **单进程复用**：每只猫同一时刻只有一个 CLI 进程，切换线程时杀旧启新
- **持久化**：session ID 写入磁盘，程序重启后可恢复
- **无冗余**：`threads/index.json` 只存线程元数据，`sessions.json` 只存 CLI session 映射，各管各的
